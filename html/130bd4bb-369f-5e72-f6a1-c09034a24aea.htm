BOOSEApp Namespace

Provides the core classes for the BOOSE graphics application, including the main user interface, command parsing/execution, drawing support, and related utilities.

Classes

AppCanvas Implements the BOOSE canvas interface using GDI+ graphics for drawing operations. Provides methods for drawing shapes, managing pen and brush properties, and manipulating the drawing surface.
AppStoredProgram Extended StoredProgram that integrates with ExecutionStateManager for proper if/else/endif handling. This class wraps the execution of commands to respect conditional execution flow.
ArrayCommand Implements the BOOSE Array command to declare arrays of integers or reals. Extends BOOSE.Array to provide compatibility with BOOSE's Peek and Poke commands. Syntax: array [type] [name] [size] [optional dimensions] Example: array int myArray 10,2 (creates 10x2 array) array real prices 10 (creates 1D array of size 10)
BOOSEApp Main form for the BOOSE graphics application. Provides a user interface for executing BOOSE commands, displaying graphics, and managing debugging output.
CallMethodCommand Represents the call method command in the BOOSE language. This command invokes a previously defined method, transferring program execution to that method and returning when it completes.
CircleCommand Represents the circle drawing command in the BOOSE language. This command is responsible for drawing a circle on the canvas using the current pen colour and position.
Clear Represents the clear command in the BOOSE language. This command removes all drawings from the canvas and resets the drawing surface to its initial blank state.
ElseCommand Implements the BOOSE Else command for conditional execution. Used after an If command to execute alternative code when the condition is false. Syntax: if condition ... else ... endif Example: if x > 5 forward 100 else back 50 endif
EndForCommand Implements the BOOSE Endfor command to mark the end of a for loop block. Used after a For command to close the loop execution block. Syntax: endfor or end for Example: for i = 1 to 10 step 1 ... endfor
EndIfCommand Represents the endif control command in the BOOSE language. This command marks the end of an if conditional block and signals the program to resume normal sequential execution.
EndMethodCommand Implements the BOOSE End Method command to mark the end of a method block. Syntax: end method
EndWhileCommand Implements the BOOSE Endwhile command to mark the end of a loop block. Used after a While command to close the loop execution block. Syntax: endwhile or end while Example: while x > 5 ... endwhile
ExecutionStateManager Manages the execution state for conditional blocks (if/else/endif). Tracks nested IF contexts and determines whether normal commands should execute.
ExecutionStateManagerConditionalContext Represents a single if/else/endif conditional block context.
ForCommand Implements the BOOSE For command to mark the beginning of a for loop block. Used to execute a block of commands repeatedly for a range of values. Syntax: for variable = start to end step increment Example: for i = 1 to 10 step 1 ... endfor
IfCommand  
IntCommand Implements the BOOSE Int command to declare integer variables. Adds custom restrictions to limit the total number of variables to no more than 100. Throws a RestrictionException if the limit is exceeded.
MethodCommand Represents the method declaration command in the BOOSE language. This command defines a reusable block of code that can be invoked later using the call command.
MethodDefinition Represents a user-defined method in the BOOSE language. This class stores metadata about a method, including its name, parameters, and the location of its executable code within the stored program.
MoveToCommand Represents the moveto drawing command in the BOOSE language. This command moves the pen to a new absolute position on the canvas without drawing a line.
MyCommandFactory Factory class for creating BOOSE command instances. Implements the Factory design pattern by extending CommandFactory to provide creation of custom commands (Clear, Reset, Circle, MoveTo, Rectangle) while delegating other commands to the base factory. Uses the singleton pattern to ensure a single factory instance throughout the application.
PeekCommand Represents the peek command in the BOOSE language. This command retrieves and displays the value stored in a variable without modifying the program state.
PokeCommand Represents the poke command in the BOOSE language. This command updates the value of an existing variable in memory, allowing data to be stored and modified during program execution.
Program  
RealCommand Represents the real variable declaration command in the BOOSE language. This command is used to declare and initialise floating-point (real) variables that can be used in expressions and calculations during program execution.
RectangleCommand Represents the rectangle drawing command in the BOOSE language. This command draws a rectangle on the canvas using the current pen position, width, height, and pen colour.
Reset Represents the reset command in the BOOSE language. This command resets the canvas to its default state by clearing all drawings and returning the pen to its initial position and settings.
WhileCommand Implements the BOOSE While command to mark the beginning of a loop block. Used to execute a block of commands repeatedly while a condition is true. Syntax: while condition Example: while x > 5 ... endwhile
WriteCommand Represents the write command in the BOOSE language. This command outputs text to the canvas or application output area, allowing messages, variable values, and program feedback to be displayed.

Structures

BOOSEAppForLoopContext Stores active for loop context (loop variable name, end value, step value). Used to manage for loop iteration.
